# Проект: Платформа менторинга (MVP)

## 1. Цель проекта

Разработать веб-приложение для менторинга.

Платформа предназначена для:
- поиска менторов по специализации и темам,
- асинхронной отправки заявок на менторство,
- установления контакта между ментором и менти.

Платформа **не является маркетплейсом** и **не является системой общения**.  
После принятия заявки дальнейшее взаимодействие происходит вне платформы.

---

## 2. Целевая аудитория

- Сотрудники компании
- Возраст: 20–45 лет
- В основном IT-специалисты
- Высокий уровень цифровой грамотности

---

## 3. Ключевые принципы

- MVP без переусложнения
- Без платных сторонних сервисов
- Без встроенного чата
- Прозрачные и простые пользовательские сценарии
- Минимально достаточная модель данных
- Только веб-приложение (responsive)

---

## 4. Роли пользователей

### Пользователь
Базовая сущность системы.

### Ментор
Пользователь, принимающий заявки на менторство.

### Менти
Пользователь, отправляющий заявки на менторство.

### Администратор
Пользователь с правами управления справочниками.

> Ограничение MVP: у пользователя может быть только одна активная роль.

---

## 5. Функциональные требования (Use Cases)

### UC-01. Регистрация пользователя
- Email + пароль
- OAuth Google (опционально)
- Валидация email
- Хеширование пароля

---

### UC-02. Авторизация
- Email + пароль или OAuth
- JWT (access + refresh)

---

### UC-03. Выбор роли
- Ментор или менти
- Одна активная роль (MVP)

---

### UC-04. Управление профилем

#### Общие поля профиля (форма UI)
- Фамилия — обязательно
- Имя — обязательно
- Отчество — необязательно
- Текущая специальность — обязательно
- Уровень (junior / middle / senior / lead) — необязательно
- Краткая информация о себе — необязательно
- Город проживания — необязательно
- Фото профиля — необязательно

Если фото отсутствует — отображается круг с первой буквой имени.

---

#### Поля профиля ментора
- Темы специализации (теги) — обязательно
- Описание услуг — обязательно
- Формат работы — обязательно
- Принимает заявки (да / нет) — обязательно
- Комментарий к статусу — обязательно, если заявки не принимаются
- Максимальное количество активных менти — обязательно
- Количество добавлений в избранное — только для отображения

---

#### Поля профиля менти
- Цель или проблема поиска ментора — обязательно
- Желаемая должность — необязательно
- Статус поиска (ищу / не ищу) — обязательно

---

### UC-05. Каталог менторов

Карточка ментора:
- Имя и фамилия
- Дата регистрации
- Специальность
- Статус приёма заявок
- Теги
- Кнопка «В избранное»
- Кнопка «Оставить заявку» (если доступно)

---

### UC-06. Фильтрация менторов
- По специальности
- По тегам
- По статусу приёма заявок

По умолчанию отображаются только менторы, принимающие заявки.

---

### UC-07. Избранное
- Менти может добавить любого ментора в избранное
- Статус ментора не влияет на добавление
- Ментор видит только общее количество добавлений

---

### UC-08. Отправка заявки
- Доступно только если ментор принимает заявки
- Обязательное сопроводительное письмо
- Статусы заявки: отправлена / принята / отклонена
- Email-уведомление ментору

---

### UC-09. Обработка заявки
- Ментор может принять или отклонить заявку
- При принятии:
  - открываются контакты
  - создаётся активная связь «ментор–менти»

---

### UC-10. Просмотр заявок
- Ментор видит входящие заявки
- Менти видит отправленные заявки

---

### UC-11. Прикрепление и открепление
- Прикрепление происходит автоматически при принятии заявки
- Открепление может инициировать любая сторона
- Причина — необязательная
- После открепления контакты скрываются

---

### UC-12. Уведомления
- Email-уведомления:
  - новая заявка
  - решение по заявке
  - открепление
- Возможность полного отключения уведомлений

---

### UC-13. Администрирование
- Управление тегами (создание / редактирование / удаление)
- Просмотр пользователей

---

## 6. Требования к базе данных

(см. сущности: users, profiles, mentors, mentees, tags, mentor_tags, favorites, requests, connections)

---

## 7. Нефункциональные требования

- Responsive web
- REST API
- HTTPS
- JWT
- OAuth 2.0 (Google)
- Время ответа API ≤ 500 мс
- Защита от XSS, CSRF, SQL-инъекций
- Возможность масштабирования

---

## 8. Контракты UI - Backend (общее описание)

- Протокол: HTTP / HTTPS
- Формат данных: JSON
- Все защищённые запросы требуют JWT
- Единый формат ошибок:
  - code
  - message

---

## 9. Language Policy

- Основной язык системы: **русский**
- Все тексты UI, email-уведомлений и ошибок — на русском языке
- Английский язык допускается:
  - в коде
  - в названиях переменных
  - в технических комментариях
- Локализация и i18n **не входят в MVP**

---

## 10. Ограничения MVP

- Нет встроенного чата
- Нет платёжных функций
- Нет мобильного приложения
- Нет расширенной аналитики
- Нет сложной админки

---

## 11. Критерии приёмки MVP

- Пользователь может зарегистрироваться и войти
- Реализован личный кабинет и профиль
- Работает каталог менторов и фильтрация
- Работает система заявок
- Реализовано избранное
- Открываются контакты при принятии заявки
- Возможна отвязка
- Реализованы светлая и тёмная темы

## 12. Технологический стек (MVP)

**Frontend:** React + TypeScript + Vite, TailwindCSS + Headless UI (или MUI), React Router, React Query  
**Backend:** Node.js + TypeScript + NestJS, Prisma ORM, JWT (access + refresh), bcrypt, nodemailer (SMTP, async queue)  
**База данных:** PostgreSQL  
**Инфраструктура:** Docker (frontend / backend / DB), Nginx для HTTPS и проксирования  
**Ограничения:** не использовать платные сервисы, Kubernetes, мобильное приложение, встроенный чат
